# Security Policy

## Supported Versions

Only the latest version on the default branch is supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public issue.**

Instead, email **security@azion.com** with:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

You should receive an initial response within **3 business days**. We will work with you to understand the issue and coordinate a fix before any public disclosure.

## Security Practices

- Dependencies are monitored via automated scanning
- Security patches are prioritized based on severity
- All code changes require peer review before merging
